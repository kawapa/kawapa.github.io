# Wyłaczenie remote-cache w bazel

Zakomentuj ten fragment w .bazelrc:

```bazel
build --remote_cache=http://replace-with-your.host:port
```

Alternatywnie można dodać flagę `--noremote_accept_cached` bezpośrednio do komendy.

# Ciekawe narzędzia

[Pre-commit](https://pre-commit.com/) - umożliwia skonfigurowanie dodatkowych check'ów przed wypushowaniem zmian do zdalnego repozytorium. Przykładowo, pre-commit może uruchamiać formatter'y kodu lub testy jednostkowe.

[PlantUML](https://plantuml.com/) - darmowe narzędzie do tworzenia różnego rozdzaju diagramów.

[K9S](https://k9scli.io/) - wygodny (terminalowy) UI do obsługi Kubernetesa. Podstawowe użycie:

```
k9s -n <NAMESPACE>
```

... i potem obsługa PODów poprzez:
- `d` (describe)
- `v` (view)
- `e` (edit)
- `l` (logs)
- `s` (shell)

Wychodzimy przez `:q` lub `Ctrl + C`.

# Usuwanie namespace'a z K8S

Rozwiązuje problem wiecznego statusu `Terminating`.
Podmień `<NAMESPACE>` oraz `<K8S_INSTANCE_IP_WITH_PORT>` na swoje.

```bash
export NS_DELETE=<NAMESPACE> && \
    kubectl get namespace ${NS_DELETE} -o json > tmp.json && \
    jq '.spec.finalizers=[]' "tmp.json" > tmp2.json && \
    mv tmp2.json "tmp.json" && \
    kubectl proxy

export NS_DELETE=<NAMESPACE> && \
    curl -k -H "Content-Type: application/json" \
    -X PUT --data-binary @tmp.json \
    <K8S_INSTANCE_IP_WITH_PORT>/api/v1/namespaces/${NS_DELETE}/finalize && \
    rm tmp.json
```
